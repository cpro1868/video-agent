- MVP 链路已明确：视频 URL 输入、字幕优先、音频 ASR 降级、LLM 摘要、标准 JSON 输出。
- 核心工程约束已明确：`stdout` 纯 JSON、日志走 `stderr`、异常映射为 JSON、临时目录默认清理。
- 发布方式已明确：`uv` 源码运行、`pipx` 安装、Wheel 包分发、Docker 容器化。

## 已确认的实施决策

### 1. Python 包结构

采用 `src/video_agent_skill/` 包结构，而不是长期把所有模块放在根目录。这样更利于 Wheel 构建、测试隔离和后续发布。

确认规则：

- CLI 入口为 `src/video_agent_skill/cli.py`。
- 控制台命令固定为 `video-agent`。
- 可选保留根目录 `main.py` 作为开发兼容入口，但正式发布入口以 `pyproject.toml` 中的 `video-agent` 为准。

### 2. 默认 LLM 提供方

文档中同时提到 Ollama、OpenAI 兼容 API、DeepSeek。已确认默认接口形态、配置读取优先级和自动切换规则。

确认规则：

- 默认使用 OpenAI 兼容接口。
- 本地默认 `api_base=http://127.0.0.1:11434/v1`，`model_name=qwen2.5`。
- 云端 API 只发送文本，不上传原始音视频。
- 配置读取优先级为：CLI 参数 > 环境变量 > `config.yaml` > 内置默认值。
- 本地 Ollama 不可用时默认不自动切换到云端 API，避免意外产生费用或数据外发；必须由用户显式配置云端接口后才调用。

### 3. ASR 运行设备策略

文档已定义 `auto`、`cuda`、`mps`、`cpu`，设备选择和失败处理规则已确认。

确认规则：

- 配置为 `auto` 时按 `cuda -> mps -> cpu` 探测。
- 显存不足时返回 `CUDA_OOM_ERROR`，不自动切到 CPU 重跑长任务，避免 Agent 等待过久。
- 显式配置 `cuda`、`mps` 或 `cpu` 时只使用指定设备；不可用则直接返回 `ASR_RUNTIME_ERROR`。
- `auto` 只在模型加载前选择设备，不在长任务失败后反复重试。

### 4. 平台支持边界

MVP 平台边界不限，但优先保证 B站和 YouTube 的端到端质量。

确认规则：

- B站和 YouTube 作为首批验收样例，必须覆盖字幕通道和 ASR 通道。
- 其他 `yt-dlp` 支持的平台可尝试解析，但不作为首批稳定性承诺。
- 付费、私有、登录受限内容只返回 `AUTH_REQUIRED_ERROR`，不做绕过。

### 5. 字幕语言匹配规则

需要明确 `zh` 与平台字幕代码的映射关系。

建议：

- `zh` 匹配 `zh`、`zh-Hans`、`zh-Hant`、`zh-CN`、`zh-TW`。
- `en` 匹配 `en`、`en-US`、`en-GB`。
- 优先人工字幕，其次自动字幕。

### 6. 输出 JSON 最终字段

最终 JSON schema 固定，避免测试和代码分歧。

固定字段：

- `status`
- `meta.url`
- `meta.strategy_used`
- `meta.language`
- `meta.duration_seconds`
- `content.summary`
- `content.key_points`
- `content.tags`
- `content.transcript_excerpt`
- `error`

成功时 `content` 必须为对象且 `error=null`；失败时 `content=null` 且 `error` 必须包含 `code` 和 `message`。
