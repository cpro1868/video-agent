## 一、配置系统概述

### 1.1 配置优先级

Video-Agent-Skill 采用**四层配置优先级**，优先级从高到低：

```
CLI 参数 > 环境变量 > config.yaml > 内置默认值
```

**示例**：
- 如果 CLI 参数 `--llm-api-key` 设置了值，它将覆盖所有其他配置
- 如果 CLI 未设置，则检查环境变量 `VIDEO_AGENT_LLM_API_KEY`
- 如果环境变量也未设置，则读取 `config.yaml` 中的 `ai.llm.api_key`
- 如果以上都未设置，使用内置默认值（仅开发测试用）

### 1.2 支持的配置项

| 配置项 | CLI 参数 | 环境变量 | config.yaml 路径 |
|--------|----------|----------|------------------|
| **代理** | `--proxy` | `VIDEO_AGENT_PROXY` | `network.default_proxy` / `network.rules` |
| **LLM API Key** | `--llm-api-key` | `VIDEO_AGENT_LLM_API_KEY` | `ai.llm.api_key` |
| **LLM API Base** | `--llm-api-base` | `VIDEO_AGENT_LLM_API_BASE` | `ai.llm.api_base` |
| **LLM Model** | `--llm-model` | `VIDEO_AGENT_LLM_MODEL` | `ai.llm.model_name` |
| **LLM Timeout** | - | `VIDEO_AGENT_LLM_TIMEOUT` | `ai.llm.timeout_seconds` |
| **LLM System Prompt** | `--llm-system-prompt` | `VIDEO_AGENT_LLM_SYSTEM_PROMPT` | `ai.llm.system_prompt` |
| **LLM User Prompt** | `--llm-user-prompt-template` | `VIDEO_AGENT_LLM_USER_PROMPT_TEMPLATE` | `ai.llm.user_prompt_template` |
| **LLM System Prompt File** | `--llm-system-prompt-file` | `VIDEO_AGENT_LLM_SYSTEM_PROMPT_FILE` | - |
| **LLM User Prompt File** | `--llm-user-prompt-file` | `VIDEO_AGENT_LLM_USER_PROMPT_TEMPLATE_FILE` | - |
| **ASR Device** | `--asr-device` | `VIDEO_AGENT_ASR_DEVICE` | `ai.asr.device` |
| **ASR Model** | - | `VIDEO_AGENT_ASR_MODEL` | `ai.asr.model` |
| **SenseVoice Source** | `--sensevoice-source-dir` | `VIDEO_AGENT_SENSEVOICE_SOURCE_DIR` | `ai.asr.source_dir` |
| **临时目录** | - | - | `system.temp_dir` |
| **自动清理** | `--keep-temp`（反向） | - | `system.auto_cleanup` |

## 二、配置测试场景

### 2.1 场景一：使用 config.yaml 配置 LLM

**目的**：验证 config.yaml 正确加载并生效

**步骤**：

```powershell
# 1. 创建测试配置文件
Set-Content -Path config.yaml -Value @"
system:
  temp_dir: ""
  auto_cleanup: true

network:
  default_proxy: ""
  rules:
    - domains: ["youtube.com", "youtu.be"]
      proxy: "http://127.0.0.1:8964"

ai:
  asr:
    device: "cpu"
    model: "iic/SenseVoiceSmall"
    source_dir: ""
  llm:
    api_base: "https://api.minimaxi.com/v1"
    model_name: "MiniMax-M2.7"
    api_key: ""
    timeout_seconds: 60
"@

# 2. 不使用任何 CLI 参数或环境变量，验证 config.yaml 生效
uv run video-agent --doctor

# 3. 检查输出中的 llm_config 部分
# 应显示：
#   - api_base: https://api.minimaxi.com/v1
#   - model_name: MiniMax-M2.7
#   - api_key: (内置默认值或空)
```

**验证点**：
- `--doctor` 输出中 `llm_config.api_base` 为 `https://api.minimaxi.com/v1`
- `llm_config.model_name` 为 `MiniMax-M2.7`

---

### 2.2 场景二：环境变量覆盖 config.yaml

**目的**：验证环境变量优先级高于 config.yaml

**步骤**：

```powershell
# 1. 确保 config.yaml 存在且设置了 api_base
# config.yaml 中 api_base: "https://api.minimaxi.com/v1"

# 2. 设置环境变量覆盖
$env:VIDEO_AGENT_LLM_API_BASE="https://api.openai.com/v1"
$env:VIDEO_AGENT_LLM_MODEL="gpt-4"

# 3. 运行 doctor 检查配置
uv run video-agent --doctor

# 4. 清理环境变量
Remove-Item Env:VIDEO_AGENT_LLM_API_BASE
Remove-Item Env:VIDEO_AGENT_LLM_MODEL
```

**验证点**：
- `llm_config.api_base` 应为 `https://api.openai.com/v1`（环境变量值）
- `llm_config.model_name` 应为 `gpt-4`（环境变量值）
- **不是** config.yaml 中的 `https://api.minimaxi.com/v1`

---

### 2.3 场景三：CLI 参数覆盖环境变量和 config.yaml

**目的**：验证 CLI 参数优先级最高

**步骤**：

```powershell
# 1. 设置环境变量
$env:VIDEO_AGENT_LLM_API_BASE="https://from-env.example.com/v1"

# 2. config.yaml 中也有 api_base: "https://from-config.example.com/v1"

# 3. 使用 CLI 参数覆盖
uv run video-agent --doctor --llm-api-base "https://from-cli.example.com/v1"

# 4. 清理环境变量
Remove-Item Env:VIDEO_AGENT_LLM_API_BASE
```

**验证点**：
- `llm_config.api_base` 应为 `https://from-cli.example.com/v1`（CLI 参数值）
- **不是** 环境变量或 config.yaml 中的值

---

### 2.4 场景四：代理路由规则测试

**目的**：验证 `network.rules` 域名匹配规则

**步骤**：

```powershell
# 1. 创建包含代理规则的配置
Set-Content -Path config.yaml -Value @"
network:
  default_proxy: "http://default-proxy:8080"
  rules:
    - domains: ["youtube.com", "youtu.be"]
      proxy: "http://youtube-proxy:8964"
    - domains: ["bilibili.com"]
      proxy: "direct"
"@

# 2. 测试不同 URL 的代理选择
uv run python -c "
from video_agent_skill.utils.config import load_config, resolve_proxy

config = load_config('config.yaml')

# YouTube 应使用 youtube-proxy
proxy = resolve_proxy('https://www.youtube.com/watch?v=test', config)
assert proxy == 'http://youtube-proxy:8964', f'YouTube proxy: {proxy}'

# B站应直连
proxy = resolve_proxy('https://www.bilibili.com/video/BV1test', config)
assert proxy == 'direct', f'Bilibili proxy: {proxy}'

# 其他域名应使用 default_proxy
proxy = resolve_proxy('https://example.com/video', config)
assert proxy == 'http://default-proxy:8080', f'Default proxy: {proxy}'

print('代理路由规则测试通过!')
"
```

**验证点**：
- YouTube URL → `http://youtube-proxy:8964`
- B站 URL → `direct`
- 其他 URL → `http://default-proxy:8080`

## 四、配置测试脚本

### 4.1 一键运行配置测试

```powershell
# 保存为 test-config.ps1

Write-Host "=== 配置系统测试 ===" -ForegroundColor Green

# 测试 1：config.yaml 加载
Write-Host "`n[CFG-001] 测试 config.yaml LLM 配置加载..."
@"
ai:
  llm:
    api_base: https://from-config.example.com/v1
    model_name: config-test-model
"@ | Set-Content -Path config.yaml

$result = uv run video-agent --doctor 2>&1 | Out-String
if ($result -match "from-config.example.com") {
    Write-Host "  PASS: config.yaml LLM 配置生效" -ForegroundColor Green
} else {
    Write-Host "  FAIL: config.yaml LLM 配置未生效" -ForegroundColor Red
}

# 测试 2：环境变量覆盖
Write-Host "`n[CFG-002] 测试环境变量覆盖 config.yaml..."
$env:VIDEO_AGENT_LLM_API_BASE = "https://from-env.example.com/v1"
$result = uv run video-agent --doctor 2>&1 | Out-String
if ($result -match "from-env.example.com") {
    Write-Host "  PASS: 环境变量覆盖成功" -ForegroundColor Green
} else {
    Write-Host "  FAIL: 环境变量未覆盖" -ForegroundColor Red
}
Remove-Item Env:VIDEO_AGENT_LLM_API_BASE

# 测试 3：CLI 参数覆盖
Write-Host "`n[CFG-003] 测试 CLI 参数覆盖..."
$result = uv run video-agent --doctor --llm-api-base "https://from-cli.example.com/v1" 2>&1 | Out-String
if ($result -match "from-cli.example.com") {
    Write-Host "  PASS: CLI 参数覆盖成功" -ForegroundColor Green
} else {
    Write-Host "  FAIL: CLI 参数未覆盖" -ForegroundColor Red
}

# 测试 4：代理路由规则
Write-Host "`n[CFG-004] 测试代理路由规则..."
uv run python -c "
from video_agent_skill.utils.config import load_config, resolve_proxy
config = load_config('config.yaml')

# 重新设置配置
import yaml
config_data = {
    'network': {
        'default_proxy': 'http://default:8080',
        'rules': [
            {'domains': ['youtube.com'], 'proxy': 'http://yt:8964'},
            {'domains': ['bilibili.com'], 'proxy': 'direct'}
        ]
    }
}

from video_agent_skill.utils.config import parse_config
c = parse_config(config_data)

tests = [
    ('https://www.youtube.com/watch?v=1', 'http://yt:8964'),
    ('https://www.bilibili.com/video/BV1', 'direct'),
    ('https://example.com/video', 'http://default:8080'),
]

for url, expected in tests:
    actual = resolve_proxy(url, c)
    status = 'PASS' if actual == expected else 'FAIL'
    print(f'  {status}: {url} -> {actual} (expected: {expected})')
"

Write-Host "`n=== 配置测试完成 ===" -ForegroundColor Green
```

## 五、常见配置问题排查

### 5.1 配置未生效

**症状**：修改了 config.yaml 但配置未生效

**排查步骤**：
```powershell
# 1. 确认 config.yaml 位置（必须在当前工作目录）
Get-ChildItem config.yaml

# 2. 确认 YAML 格式正确
uv run python -c "import yaml; yaml.safe_load(open('config.yaml'))"

# 3. 检查是否有环境变量覆盖
Get-ChildItem Env: | Where-Object { $_.Name -like "VIDEO_AGENT*" }

# 4. 使用 --doctor 确认最终生效配置
uv run video-agent --doctor
```

### 5.2 代理配置无效

**症状**：设置了代理但网络请求失败

**排查步骤**：
```powershell
# 1. 确认代理服务器可用
curl -x "http://127.0.0.1:8964" https://www.youtube.com -I

# 2. 确认代理规则匹配
uv run python -c "
from video_agent_skill.utils.config import load_config, resolve_proxy
config = load_config()
print(f'YouTube proxy: {resolve_proxy(\"https://www.youtube.com/watch?v=1\", config)}')
"

# 3. 确认代理协议正确（http/socks5）
# config.yaml 中应该是 "http://..." 或 "socks5://..."
```

### 5.3 API Key 泄露

**症状**：担心 API Key 被意外记录

**排查步骤**：
```powershell
# 1. 确认 config.yaml 中没有明文 Key
Get-Content config.yaml | Select-String "api_key"

# 2. 确认环境变量是否意外设置
Get-ChildItem Env: | Where-Object { $_.Name -like "*API_KEY*" }

# 3. 使用 CLI 参数传递而非写入文件
# 推荐：uv run video-agent --llm-api-key "sk-xxx"
# 不推荐：写入 config.yaml
```

## 六、配置最佳实践

### 6.1 生产环境推荐配置

```yaml
# config.yaml - 生产环境配置示例
system:
  temp_dir: "/var/tmp/video-agent"  # 使用系统临时目录
  auto_cleanup: true                 # 自动清理临时文件

network:
  default_proxy: ""                  # 无默认代理
  rules:
    - domains: ["youtube.com", "youtu.be"]
      proxy: "socks5://proxy.internal:7890"  # 内网代理
    - domains: ["bilibili.com"]
      proxy: "direct"                        # 直连

ai:
  asr:
    device: "cuda"                    # 使用 GPU
    model: "iic/SenseVoiceSmall"
    source_dir: "/opt/sensevoice"      # 共享模型目录
  llm:
    api_base: "https://api.minimaxi.com/v1"
    model_name: "MiniMax-M2.7"
    # api_key 不写入文件，使用环境变量或 CLI 参数
    timeout_seconds: 120
```

### 6.2 安全建议

1. **不要在 config.yaml 中写入敏感信息**（API Key、密码等）
2. **使用环境变量传递敏感配置**
3. **生产环境使用 CLI 参数或密钥管理服务**
4. **不要将 config.yaml 提交到版本控制**（添加到 .gitignore）
