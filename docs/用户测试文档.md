## 一、测试前准备

### 1.1 环境要求

| 项目 | 要求 | 检查命令 |
|------|------|----------|
| Python | 3.10+ | `python --version` |
| Git | 任意版本 | `git --version` |
| FFmpeg | 最新稳定版 | `ffmpeg -version` |
| 网络代理 | 访问 YouTube 需要 | 确认代理可用 |
| LLM API Key | MiniMax 或其他 OpenAI 兼容 API | 准备测试 Key |

### 1.2 获取测试版本

```powershell
# 1. 克隆仓库
git clone <仓库地址> video-agent-skill
cd video-agent-skill

# 2. 确认当前目录结构
ls
# 应看到：src/、tests/、docs/、pyproject.toml、config.example.yaml 等
```

### 1.3 快速环境检查

```powershell
# 运行环境诊断（不需要 URL）
uv run video-agent --doctor
```

**预期输出**：JSON 格式，显示各组件状态：
- `yt_dlp`：✅ available
- `ffmpeg`：✅ available
- `asr_device`：cpu / cuda / mps（根据你的硬件）
- `llm_config`：显示当前 LLM 配置

## 三、详细测试用例

### TC-001：环境诊断

**目的**：验证所有依赖组件已就绪

**命令**：
```powershell
uv run video-agent --doctor
```

**预期结果**：
- 退出码：0
- stdout：JSON 格式，包含各组件状态
- 关键字段：
  - `status` = "success"
  - `yt_dlp.available` = true
  - `ffmpeg.available` = true

**通过标准**：所有关键组件显示 available

---

### TC-002：YouTube 字幕提取

**目的**：验证字幕优先通道正常工作

**前置条件**：需要可用代理访问 YouTube

**命令**：
```powershell
uv run video-agent -u "https://www.youtube.com/watch?v=KGUXXUCV6S4" --lang en --proxy "http://127.0.0.1:8964" --transcript-only
```

**预期结果**：
- `status` = "success"
- `meta.strategy_used` = "subtitle"
- `meta.language` = "en"
- `content.text` 非空，包含英文字幕内容

**验证方法**：
```powershell
# 将输出保存到文件
uv run video-agent -u "https://www.youtube.com/watch?v=KGUXXUCV6S4" --lang en --proxy "http://127.0.0.1:8964" --transcript-only > tc002_output.json

# 验证 JSON 可解析
uv run python -c "import json; d=json.load(open('tc002_output.json')); assert d['status']=='success'; assert d['meta']['strategy_used']=='subtitle'; print('TC-002 PASS')"
```

### TC-003：YouTube 音频 ASR 降级

**目的**：验证无字幕视频自动降级到 ASR

**前置条件**：需要可用代理 + ASR 环境（CPU 即可）

**命令**：
```powershell
uv run video-agent -u "https://www.youtube.com/watch?v=eV6gHhliroA" --lang en --proxy "http://127.0.0.1:8964" --transcript-only
```

**预期结果**：
- `status` = "success"
- `meta.strategy_used` = "asr"
- `content.text` 非空，包含 ASR 转写文本
- stderr 中可能显示 ASR 加载和推理日志

**注意**：ASR 首次运行可能需要下载模型，耗时较长（5-10 分钟）

### TC-004：LLM 摘要生成

**目的**：验证完整链路（提取 → LLM 摘要 → JSON 输出）

**前置条件**：需要配置 LLM API Key

**命令**：
```powershell
# 方式 1：环境变量
$env:VIDEO_AGENT_LLM_API_KEY="your-minimax-api-key"
uv run video-agent -u "https://www.youtube.com/watch?v=KGUXXUCV6S4" --lang zh --proxy "http://127.0.0.1:8964"

# 方式 2：命令行参数
uv run video-agent -u "https://www.youtube.com/watch?v=KGUXXUCV6S4" --lang zh --proxy "http://127.0.0.1:8964" --llm-api-key "your-minimax-api-key"
```

**预期结果**：
- `status` = "success"
- `content.summary` 非空，中文，约 200 字
- `content.key_points` 包含 3-5 条核心观点
- `content.tags` 包含相关标签

**验证方法**：
```powershell
# 检查 summary 长度
uv run python -c "
import json
d = json.load(open('tc004_output.json'))
assert d['status'] == 'success'
assert len(d['content']['summary']) > 50
assert len(d['content']['key_points']) >= 3
print('TC-004 PASS')
"
```

Get-Content tc005_stderr.log | Select-Object -First 5

# 3. stdout 文件不应包含非 JSON 内容
uv run python -c "

### TC-013：配置文件加载与优先级

**目的**：验证 config.yaml 加载和配置优先级（CLI > 环境变量 > config.yaml > 默认值）

**前置条件**：需要了解配置系统优先级

**步骤**：

```powershell
# 步骤 1：创建测试配置文件
Set-Content -Path config.yaml -Value @"
ai:
  llm:
    api_base: "https://from-config.example.com/v1"
    model_name: "config-model"
"@

# 步骤 2：验证 config.yaml 生效
uv run video-agent --doctor
# 预期：api_base = "https://from-config.example.com/v1"

# 步骤 3：设置环境变量覆盖
$env:VIDEO_AGENT_LLM_API_BASE="https://from-env.example.com/v1"
uv run video-agent --doctor
# 预期：api_base = "https://from-env.example.com/v1"

# 步骤 4：使用 CLI 参数覆盖
uv run video-agent --doctor --llm-api-base "https://from-cli.example.com/v1"
# 预期：api_base = "https://from-cli.example.com/v1"

# 清理环境变量
Remove-Item Env:VIDEO_AGENT_LLM_API_BASE
```

**验证方法**：
```powershell
# 使用 Python 脚本验证优先级
uv run python -c "
from video_agent_skill.utils.config import load_config, resolve_llm_config, CliOverrides
import os

# 创建测试配置
config = load_config()

# 验证优先级：CLI > 环境变量 > config.yaml
# 1. 仅 config.yaml
llm = resolve_llm_config(config)
print(f'config.yaml only: {llm.api_base}')

# 2. 环境变量覆盖
os.environ['VIDEO_AGENT_LLM_API_BASE'] = 'https://from-env.example.com/v1'
llm = resolve_llm_config(config)
print(f'with env: {llm.api_base}')

# 3. CLI 参数覆盖
llm = resolve_llm_config(config, CliOverrides(llm_api_base='https://from-cli.example.com/v1'))
print(f'with CLI: {llm.api_base}')

del os.environ['VIDEO_AGENT_LLM_API_BASE']
"
```

**预期结果**：
- 仅 config.yaml：`api_base` = `https://from-config.example.com/v1`
- 环境变量覆盖：`api_base` = `https://from-env.example.com/v1`
- CLI 参数覆盖：`api_base` = `https://from-cli.example.com/v1`

## 六、测试注意事项

### 6.1 网络相关

- YouTube 测试**必须**使用代理
- B站测试**暂时跳过**（已知 HTTP 412 问题）
- 代理地址请根据实际情况修改（示例中使用 `http://127.0.0.1:8964`）

### 6.2 API Key 相关

- **不要在仓库中提交真实 API Key**
- 测试完成后建议删除环境变量中的 Key
- 如使用 `--llm-api-key` 参数，命令会记录在 shell history 中

### 6.3 ASR 相关

- 首次运行 ASR 需要下载 SenseVoice 模型（约 1GB）
- CPU 推理较慢，10 分钟视频可能需要 5-10 分钟
- GPU 推理快 5-10 倍，但需要 CUDA 环境

### 6.4 输出编码

- Windows PowerShell 可能显示 UTF-8 乱码
- 建议重定向到文件后查看：`> output.json`
- 或在 PowerShell 中设置：`[Console]::OutputEncoding = [System.Text.Encoding]::UTF8`
