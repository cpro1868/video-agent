## 一、产品简介

**Video-Agent-Skill** 是一个面向 AI Agent 的无头（Headless）视频内容解析与摘要 CLI 工具。它接收流媒体视频链接，优先提取平台字幕；当字幕不可用时，降级下载低码率音频并调用本地 ASR 转写，最后输出适合 Agent 消费的结构化 JSON 或人类可读的 Markdown。

### 核心能力

| 能力 | 说明 |
|------|------|
| **字幕优先** | 优先获取 UP 主字幕或平台自动字幕，避免不必要的音频下载和本地推理 |
| **本地 ASR 降级** | 无字幕时提取音频，使用 `SenseVoiceSmall` 完成本地离线转写 |
| **弹幕分析（B站）** | 提取并分析 B站弹幕，洞察大众情感、热点话题和观众画像 |
| **双格式输出** | 支持 JSON（Agent 解析）和 Markdown（人类阅读）两种输出格式 |
| **Agent 级 I/O 契约** | 运行日志、进度条和第三方库输出全部进入 `stderr`，`stdout` 仅输出业务数据 |
| **代理路由** | 按域名匹配代理策略，适配 YouTube、B站、抖音等平台的网络差异 |
| **标准交付** | 支持 `uv` 开发运行、`pip`/`pipx` 安装和 Wheel 离线分发 |

## 二、快速开始

### 前置条件

1. **Python 3.10+**
2. **FFmpeg**（用于音频转码，请确保在 PATH 中）
3. **网络代理**（如需访问 YouTube）
4. **LLM API Key**（如需 LLM 摘要功能）

### 安装

```powershell
# 方式一：pip 安装 Wheel 包
pip install dist/video_agent_skill-*.whl

# 方式二：pipx 全局安装
pipx install .

# 方式三：uv 源码运行（开发调试）
uv run video-agent --help
```

### 首次使用

```powershell
# 1. 查看版本
video-agent --version

# 2. 环境诊断（不需要 URL）
video-agent --doctor

# 3. 编辑配置文件（填入 API Key、代理等）
# 配置文件位置：site-packages/video_agent_skill/config.yaml

# 4. 运行
video-agent -u "https://www.youtube.com/watch?v=xxxx" --lang zh
```

## 四、配置说明

### 4.1 配置加载规则

**配置只从包内 `config.yaml` 读取，CLI 参数覆盖对应字段。**

- 配置文件位置：`site-packages/video_agent_skill/config.yaml`（随包安装）
- pip uninstall 会自动清理该文件（在 wheel RECORD 里）
- CLI 参数（如 `--proxy`、`--llm-api-key`）覆盖 config.yaml 里的对应字段
- **不读环境变量、不读当前目录**

### 4.2 配置文件位置

配置文件随包安装在 Python 环境的 site-packages 目录下：

```powershell
# 查看实际路径
python -c "import video_agent_skill; from pathlib import Path; print(Path(video_agent_skill.__file__).parent / 'config.yaml')"
```

典型路径：
- Windows: `D:\Soft\anaconda3\envs\<环境名>\Lib\site-packages\video_agent_skill\config.yaml`
- Linux/Mac: `~/.local/lib/python3.10/site-packages/video_agent_skill/config.yaml`

### 4.3 配置示例

编辑 `config.yaml`：

```yaml
system:
  temp_dir: ""
  auto_cleanup: true

network:
  default_proxy: ""
  rules:
    - domains: ["youtube.com", "youtu.be"]
      proxy: "http://127.0.0.1:8964"
    - domains: ["bilibili.com", "douyin.com"]
      proxy: "direct"

ai:
  asr:
    device: "auto"
    model: "iic/SenseVoiceSmall"
    source_dir: "G:/Projects/Sources/SenseVoice"
  llm:
    api_base: "https://api.minimaxi.com/v1"
    model_name: "MiniMax-M2.7"
    api_key: ""  # 在这里填入你的 API Key
    timeout_seconds: 60
    system_prompt: ""
    user_prompt_template: ""

output:
  format: "markdown"  # json 或 markdown
  file: ""
  batch_separator: "---"
```

### 4.4 API Key 配置

在 `config.yaml` 的 `ai.llm.api_key` 字段填入 API Key：

```yaml
ai:
  llm:
    api_key: "your-api-key-here"
```

或通过 CLI 参数传入（覆盖 config.yaml）：

```powershell
video-agent -u "<URL>" --lang zh --llm-api-key "your-api-key"
```

### 4.5 CLI 参数覆盖

CLI 参数会覆盖 config.yaml 里的对应字段：

```powershell
# 覆盖代理
video-agent -u "..." --proxy "http://127.0.0.1:8964"

# 覆盖 API Key
video-agent -u "..." --llm-api-key "your-key"

# 覆盖输出格式
video-agent -u "..." --output-format json

# 覆盖 ASR 设备
video-agent -u "..." --asr-device cuda
```

## 五、命令行参数详解

| 参数 | 必填 | 说明 | 示例 |
|------|------|------|------|
| `-u, --url` | 是 | 目标视频 URL | `-u "https://www.youtube.com/watch?v=xxxx"` |
| `-l, --lang` | 否 | 优先字幕语言，默认 `zh` | `--lang en` |
| `--proxy` | 否 | 单次运行强制代理 | `--proxy "http://127.0.0.1:8964"` |
| `--keep-temp` | 否 | 保留临时文件用于调试 | `--keep-temp` |
| `--transcript-only` | 否 | 仅返回转写文本，不调用 LLM | `--transcript-only` |
| `--output-format` | 否 | `json`（默认）或 `markdown` | `--output-format markdown` |
| `--output-file` | 否 | 输出文件路径，默认 stdout | `--output-file summary.md` |
| `--llm-api-key` | 否 | LLM API Key | `--llm-api-key "sk-xxxx"` |
| `--llm-api-base` | 否 | LLM API Base URL | `--llm-api-base "https://api.minimaxi.com/v1"` |
| `--llm-model` | 否 | LLM 模型名称 | `--llm-model "MiniMax-M2.7"` |
| `--llm-system-prompt-file` | 否 | 系统 Prompt 文件路径 | `--llm-system-prompt-file "prompts/default-system.txt"` |
| `--llm-user-prompt-file` | 否 | 用户 Prompt 文件路径 | `--llm-user-prompt-file "prompts/default-video-summary.txt"` |
| `--asr-device` | 否 | ASR 设备：`auto`/`cuda`/`mps`/`cpu` | `--asr-device cuda` |
| `--sensevoice-source-dir` | 否 | SenseVoice 源码目录 | `--sensevoice-source-dir "G:/Projects/Sources/SenseVoice"` |
| `--include-danmaku` | 否 | 启用弹幕分析（仅 B站） | `--include-danmaku` |
| `--danmaku-prompt-file` | 否 | 弹幕分析 Prompt 文件路径 | `--danmaku-prompt "prompts/danmaku.txt"` |
| `--danmaku-output` | 否 | 弹幕分析独立输出文件 | `--danmaku-output "danmaku.md"` |
| `--batch` | 否 | 从文件批量处理 URL | `--batch urls.txt` |
| `--no-cache` | 否 | 禁用本次运行的结果缓存 | `--no-cache` |
| `--clear-cache` | 否 | 清理所有缓存并退出 | `--clear-cache` |
| `--no-progress-bar` | 否 | 禁用可视化进度条 | `--no-progress-bar` |
| `--doctor` | 否 | 环境诊断，不需要 URL | `--doctor` |
| `--version, -V` | 否 | 查看版本号 | `--version` |
| `--prompt-info` | 否 | 查看包内默认 Prompt 信息 | `--prompt-info` |
| `--init-prompts [DIR]` | 否 | 复制默认 Prompt 到本地目录 | `--init-prompts ./prompts` |
| `--init-config [DIR]` | 否 | 复制配置模板到指定目录 | `--init-config .` |
| `--config` | 否 | 指定 config.yaml 路径 | `--config /path/to/config.yaml` |
| `--overwrite` | 否 | 允许覆盖现有文件 | `--overwrite` |

## 六、输出格式控制

### 6.1 JSON 输出（默认）

```powershell
video-agent -u "..." --lang zh
```

输出标准 JSON 对象到 stdout，适合 Agent 解析。

### 6.2 Markdown 输出

```powershell
# Markdown 输出到 stdout
video-agent -u "..." --lang zh --output-format markdown

# Markdown 输出到文件
video-agent -u "..." --lang zh --output-format markdown --output-file summary.md
```

Markdown 输出包含：标题、视频信息、摘要、关键要点、详细内容、标签、转录片段。

### 6.3 文件输出

```powershell
# JSON 写入文件
video-agent -u "..." --lang zh --output-file result.json

# Markdown 写入文件
video-agent -u "..." --lang zh --output-format markdown --output-file summary.md
```

也可以在 `config.yaml` 里设置默认输出格式和文件：

```yaml
output:
  format: "markdown"
  file: ""
```

## 七、弹幕分析功能（B站专属）

### 7.1 功能说明

弹幕分析功能仅支持 B站视频。启用后，工具会：
1. 提取视频弹幕
2. 筛选代表性弹幕（按权重排序，去重）
3. 使用 LLM 分析弹幕情感、热点话题、观众画像
4. 将弹幕分析结果整合到视频摘要中

### 7.2 使用示例

```powershell
# 弹幕整合到主输出
video-agent -u "https://www.bilibili.com/video/BVxxxx" --lang zh --include-danmaku

# 独立输出弹幕分析文件
video-agent -u "https://www.bilibili.com/video/BVxxxx" --lang zh --include-danmaku --danmaku-output "danmaku.md"
```

## 十一、高级功能

### 11.1 结果缓存

自动缓存提取的字幕/转写文本（不缓存音频/视频文件），避免重复处理。

```powershell
# 禁用缓存
video-agent -u "..." --no-cache

# 清理所有缓存
video-agent --clear-cache
```

### 11.2 进度反馈

长视频处理时，stderr 输出进度信息。默认显示可视化进度条，用 `--no-progress-bar` 关闭。

### 11.3 错误重试

LLM 请求失败时自动重试（最多 3 次，指数退避）。

### 11.4 批量处理

```powershell
# urls.txt 每行一个 URL，# 开头为注释
video-agent --batch urls.txt --lang zh --output-file results.json
```

## 十二、常见问题（FAQ）

### Q1: stdout 输出乱码？

Windows 默认编码为 GBK，执行：
```powershell
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
```

### Q2: 如何查看版本？

```powershell
video-agent --version
```

### Q3: ASR 需要 GPU 吗？

不需要。`device: auto` 会自动探测 `cuda -> mps -> cpu`。无 GPU 自动降级到 CPU。

### Q4: LLM 摘要返回安全拒绝？

MiniMax 等 LLM 对敏感内容会返回 `LLM_SAFETY_REFUSAL`。这是 LLM 服务方限制，换非敏感视频或换 LLM 模型。

### Q5: B站视频能解析吗？

能。B站 HTTP 412 反爬问题已通过动态 buvid3/buvid4 cookie + Origin 请求头修复。B站若仅暴露 `danmaku`，不视为可用字幕，自动降级到 ASR。

### Q6: 如何自定义 Prompt？

```powershell
video-agent --prompt-info              # 查看默认 Prompt
video-agent --init-prompts ./prompts   # 复制到本地
# 编辑后使用
video-agent -u "..." --llm-user-prompt-file "./prompts/default-video-summary.txt"
```
