---

## 一、背景与目标

### 1.1 现状

Video-Agent-Skill v1.4.0 已稳定发布，支持：

- 视频字幕/音频下载（依赖 yt-dlp 覆盖 1600+ 站点）
- 字幕优先 + ASR 降级
- LLM 摘要（Markdown/JSON 双格式）
- B站弹幕分析
- SenseVoice 本地 ASR

但以下能力**硬编码在主项目中**，第三方无法扩展：

| 能力 | 现状 | 局限 |
|------|------|------|
| 弹幕抓取 | 只支持 B站（`danmaku.py` 硬编码 B站弹幕 XML） | 无法支持 YouTube 评论、AcFun 弹幕等 |
| 反爬处理 | 只支持 B站 412 绕过（`extractor.py` 硬编码） | 其他站点的反爬无法定制 |
| ASR 引擎 | 只支持 SenseVoice/FunASR（`transcriber.py` 硬编码） | 无法切换 Whisper、Paraformer 等 |

### 1.2 目标

v2.0 引入**第三方插件机制**，允许社区开发者通过独立的 PyPI 包扩展以下能力：

1. **弹幕插件点**：支持新站点的弹幕抓取
2. **反爬插件点**：支持新站点的反爬绕过
3. **ASR 引擎插件点**：支持切换不同的 ASR 引擎

设计原则：**加新站点不需要改主项目代码**。

```python
class XiaohongshuDanmakuProvider(DanmakuProvider):
    name = "xiaohongshu"

    def supports(self, url: str) -> bool:
        return "xiaohongshu.com" in url

    def extract(self, url: str, *, proxy: str = "") -> list[DanmakuItem]:
        # 实现小红书弹幕抓取
        return [...]
```

创建 `pyproject.toml`：

```toml
[project]
name = "video-agent-skill-plugin-xiaohongshu"
version = "0.1.0"
dependencies = ["video-agent-skill>=2.0"]

[project.entry-points."video_agent_skill.danmaku"]
xiaohongshu = "video_agent_skill_plugin_xiaohongshu:XiaohongshuDanmakuProvider"
```

发布：

```bash
pip install video-agent-skill-plugin-xiaohongshu
```

使用：

```bash
video-agent -u "https://xiaohongshu.com/video/xxx" --include-danmaku
# 自动调用 XiaohongshuDanmakuProvider
```

### 场景 2：第三方支持新 ASR 引擎

开发者想集成 Whisper：

```python
# video_agent_skill_plugin_whisper/asr.py
from video_agent_skill.core.transcriber import AsrEngine


class WhisperEngine(AsrEngine):
    name = "whisper"

    def transcribe(self, audio_path: str, *, device: str = "auto") -> str:

        model = whisper.load_model("base", device=self._resolve_device(device))
```

## 三、插件点详细设计

### 3.1 弹幕插件点（Danmaku Provider）

**接口定义**（`video_agent_skill/core/danmaku.py`）：

```python
from typing import Protocol, runtime_checkable

@runtime_checkable
class DanmakuProvider(Protocol):
    """弹幕提供者接口。

    第三方实现此接口以支持新站点的弹幕抓取。
    """

    name: str  # 插件唯一标识

    def supports(self, url: str) -> bool:
        """判断当前 provider 是否能处理该 URL。"""
        ...

    def extract(self, url: str, *, proxy: str = "") -> list[DanmakuItem]:
        """从 URL 抓取弹幕列表。

        Args:
            url: 视频 URL
            proxy: 可选代理

        Returns:
            DanmakuItem 列表

        Raises:
            DanmakuExtractionError: 抓取失败
        """
        ...
```

**注册方式**：通过 entry point `video_agent_skill.danmaku`。

**主项目适配**：将 `danmaku.py` 中现有 B站逻辑抽成 `BilibiliDanmakuProvider`，作为内置默认 provider。

### 3.2 反爬插件点（Anti-Crawler Handler）

**接口定义**（`video_agent_skill/core/anticrawler.py` 新建文件）：

```python
from typing import Protocol, runtime_checkable

@runtime_checkable
class AntiCrawlerHandler(Protocol):
    """反爬处理接口。

    第三方实现此接口以处理特定站点的反爬机制（如登录、token、特殊 headers）。
    """

    name: str

    def supports(self, url: str) -> bool:
        """判断当前 handler 是否适用于该 URL。"""
        ...

    def apply(self, ydl_opts: dict) -> dict:
        """修改 yt-dlp 选项 dict 以应用反爬处理。

        Args:
            ydl_opts: 当前 yt-dlp 选项

        Returns:
            修改后的 yt-dlp 选项（必须返回新 dict，不要修改原 dict）
        """
        ...
```

**注册方式**：通过 entry point `video_agent_skill.anticrawler`。

**主项目适配**：将 B站 412 绕过（`extractor.py` 中 bilibili cookie 生成和 headers 注入）抽成 `BilibiliAntiCrawlerHandler`。

### 3.3 ASR 引擎插件点（ASR Engine）

**接口定义**（`video_agent_skill/core/transcriber.py`）：

```python
from typing import Protocol, runtime_checkable

@runtime_checkable
class AsrEngine(Protocol):
    """ASR 引擎接口。

    第三方实现此接口以接入不同的 ASR 引擎（Whisper、Paraformer、Azure 等）。
    """

    name: str  # 引擎唯一标识，如 "sensevoice"、"whisper"

    def transcribe(
        self,
        audio_path: str,
        *,
        device: str = "auto",
    ) -> str:
        """转写音频文件为文本。

        Args:
            audio_path: 音频文件绝对路径（wav 格式）
            device: 设备选择 ("auto", "cuda", "mps", "cpu")

        Returns:
            转写后的纯文本

        Raises:
            AsrRuntimeError: 转写失败
            CudaOomError: GPU 显存不足
        """
        ...
```

**注册方式**：通过 entry point `video_agent_skill.asr`。

**主项目适配**：将现有 SenseVoice 逻辑抽成 `SenseVoiceEngine`，作为内置默认引擎。

### 3.4 插件发现机制

通过 Python 标准 `importlib.metadata` 的 entry_points：

```python
# video_agent_skill/core/plugin_registry.py（新建）

import logging
from importlib.metadata import entry_points
from typing import Type

from video_agent_skill.core.danmaku import DanmakuProvider, DanmakuItem
# 其他接口导入

logger = logging.getLogger(__name__)


class PluginRegistry:
    """插件注册中心（单例）。"""

    _instance: "PluginRegistry | None" = None

    def __init__(self) -> None:
        self._danmaku_providers: list[DanmakuProvider] = []
        self._anticrawler_handlers: list[AntiCrawlerHandler] = []
        self._asr_engines: dict[str, AsrEngine] = {}

    @classmethod
    def instance(cls) -> "PluginRegistry":
        if cls._instance is None:
            cls._instance = cls()
            cls._instance._load_all()
        return cls._instance

    def _load_all(self) -> None:
        """扫描所有 entry points 并加载插件。"""
        try:
            eps = entry_points(group="video_agent_skill.danmaku")
            for ep in eps:
                try:
                    provider = ep.load()()
                    self._danmaku_providers.append(provider)
                    logger.info(f"Loaded danmaku provider: {ep.name}")
                except Exception as exc:
                    logger.error(f"Failed to load danmaku plugin {ep.name}: {exc}")
        except Exception as exc:
            logger.warning(f"entry_points scan failed: {exc}")

        # 类似加载 anticrawler 和 asr 引擎
        # ...

    def get_danmaku_provider(self, url: str) -> DanmakuProvider | None:
        """根据 URL 找到匹配的弹幕提供者（按 priority 排序，取最高）。"""
        matches = [p for p in self._danmaku_providers if p.supports(url)]
        if not matches:
            return None
        # 默认按注册顺序，priority 字段未来扩展
        return matches[0]

    def get_asr_engine(self, name: str) -> AsrEngine | None:
        """根据名称获取 ASR 引擎。"""
        return self._asr_engines.get(name)
```

**加载时机**：第一次访问 `PluginRegistry.instance()` 时加载，后续缓存。

**失败处理**：单个插件加载失败不影响其他插件，记录错误日志继续。

## 四、配置与 CLI

### 4.1 配置文件

`config.yaml` 扩展：

```yaml
ai:
  asr:
    engine: "sensevoice"  # 默认引擎
    # 不同引擎的不同配置
    sensevoice:
      model: "iic/SenseVoiceSmall"
      source_dir: "G:/Projects/Sources/SenseVoice"
    # whisper 插件的示例（用户装了插件才有这些选项）
    # whisper:
    #   model: "base"
    #   device: "auto"
```

### 4.2 CLI 参数

```bash
# 选择 ASR 引擎
video-agent -u "..." --asr-engine whisper

# 强制使用某个弹幕 provider
video-agent -u "..." --include-danmaku --danmaku-provider xiaohongshu

# 跳过反爬处理（调试用）
video-agent -u "..." --no-anticrawler
```

新增参数：

- `--asr-engine ENGINE`：选择 ASR 引擎（默认 `sensevoice`）
- `--danmaku-provider NAME`：指定弹幕 provider
- `--no-anticrawler`：禁用所有反爬插件

### 4.3 插件管理命令

```bash
# 列出已安装的插件
video-agent --list-plugins

# 输出示例：
# Danmaku providers:

#   - bilibili-412 (built-in)

#   - sensevoice (built-in)
```

## 五、实现步骤（三步走）

### 步骤 1：弹幕插件化（最小可行版本）

**目标**：验证插件机制可行性，最简单的一种插件点。

**范围**：
- 新增 `video_agent_skill/core/danmaku.py` 中的 `DanmakuProvider` Protocol
- 抽离现有 B站逻辑为 `BilibiliDanmakuProvider`（内置）
- 新增 `PluginRegistry`（最小版，只支持弹幕）
- 新增 `--list-plugins` 命令
- 写一个示例第三方插件 `video-agent-skill-plugin-xiaohongshu`（仅 demo，不强制要求发布）

**预估工作量**：300-400 行代码 + 200 行测试 + 100 行示例插件

### 步骤 2：反爬插件化

**目标**：让第三方能处理特殊站点的反爬。

**范围**：
- 新增 `video_agent_skill/core/anticrawler.py`（新建文件）中的 `AntiCrawlerHandler` Protocol
- 抽离 B站 412 绕过为 `BilibiliAntiCrawlerHandler`（内置）
- 扩展 `PluginRegistry` 支持反爬
- 集成到 `extract_text_from_url`（在调用 yt-dlp 前应用所有匹配的 handler）

**预估工作量**：200-300 行代码 + 100 行测试

### 步骤 3：ASR 引擎插件化

**目标**：让用户能切换不同的 ASR 引擎。

**范围**：
- 抽离 `transcriber.py` 中 SenseVoice 逻辑为 `SenseVoiceEngine`（内置）
- 新增 `AsrEngine` Protocol
- 扩展 `PluginRegistry` 支持 ASR
- 集成到 `transcribe_audio`（根据 `--asr-engine` 选择引擎）
- 写一个 Whisper 插件示例（可选）

**预估工作量**：400-500 行代码 + 150 行测试

## 六、向后兼容

### 6.1 现有 API 保持不变

- `extract_text_from_url()`、`transcribe_audio()`、`extract_danmaku()` 函数签名不变
- `AsrConfig`、`SenseVoiceOptions` 等 dataclass 保持
- CLI 命令行参数 `video-agent -u ...` 用法不变
- 现有配置 `config.yaml` 格式无需修改

### 6.2 内置插件与第三方插件的优先级

- 内置插件（bilibili、B站反爬、sensevoice）**永远优先**于第三方
- 第三方插件作为补充
- 同一类型多个插件时，按 `priority` 字段排序（默认 0，越大越优先）

### 6.3 插件接口版本控制

- v2.0 引入 `Plugin API v1`
- 主版本号变更时，接口可能不兼容
- 破坏性变更会在 `CHANGELOG.md` 明确标注

## 七、安全与风险

### 7.1 插件安全风险

第三方插件以 `transcribe()` 任意代码执行权限运行，**主项目不做沙箱**。

**缓解措施**：
- `pip install` 时用户已确认信任插件
- README 明确说明风险
- 推荐只安装知名开发者的插件

### 7.2 失败隔离

单个插件加载失败不应影响主程序：

```python
try:
    plugin = ep.load()()
    registry.register(plugin)
except Exception as exc:
    logger.error(f"Plugin {ep.name} failed to load: {exc}")
    # 继续加载其他插件
```

### 7.3 配置验证

插件配置应该与主配置分离校验：

```python
# 插件配置校验失败，只禁用该插件，其他功能正常
try:
    engine = WhisperEngine(config.ai.asr.whisper)
    registry.register("whisper", engine)
except ConfigurationError as exc:
    logger.warning(f"Whisper plugin config invalid: {exc}")
```

## 十、开放问题（已确认）

~~需要用户/团队决定：~~

✅ **已确认决策**（2026-06-26）：

1. **插件发布到 PyPI 的策略**：
   - ✅ **完全自由**：第三方自由发布，主项目不提供官方索引
   - 主项目只提供插件开发文档和示例

2. **插件配置错误时的行为**：
   - ✅ **仅警告禁用**：启动时记录警告并禁用该插件，其他功能正常
   - 配置错误不阻断主程序运行

3. **插件签名验证**：
   - ✅ **需要签名验证**：主项目应实现插件签名机制
   - 防止恶意插件伪装成官方或知名插件

4. **插件文档规范**：
   - 强制要求 README 说明插件功能
   - 强制要求版本号和兼容的 v-agent 版本
   - 强制要求 `plugin_manifest.json` 描述插件元信息

5. **回退机制**：
   - ✅ **回退到内置默认**：如果所有第三方插件都不可用，自动回退到内置默认
   - 内置插件（bilibili、B站反爬、sensevoice）作为 fallback
