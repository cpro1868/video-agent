### 4.2 已知问题（来自开发团队）

#### Issue-001：B站 HTTP 412 反爬问题

- **发现时间**：2026-06-10
- **发现人**：开发团队
- **关联测试用例**：TC-XXX（B站相关用例）
- **严重程度**：P0
- **问题描述**：B站视频解析返回 HTTP 412 Precondition Failed
- **复现步骤**：
  1. 执行 `uv run video-agent -u "https://www.bilibili.com/video/BV1XDLn6XEWK" --lang zh`
  2. 观察输出
- **预期结果**：正常解析并返回 JSON
- **实际结果**：返回 HTTP 412 错误
- **环境信息**：所有环境均可复现
- **状态**：✅ **已修复**（2026-06-17）
- **根因分析**：
  - B站升级反爬策略，要求请求包含 `buvid3`/`buvid4` 指纹 cookie 和 `Origin` 请求头
  - 原 yt-dlp 代码的 `BilibiliBaseIE._HEADERS` 仅包含 `Referer`，缺少 `Origin`
- **修复方案**（已实现于 `src/video_agent_skill/core/extractor.py`）：
  - 新增 `_get_bilibili_cookie_file()` 函数：动态生成包含 `buvid3`/`buvid4` 指纹的 Netscape cookie 文件
  - 实现逻辑：调用 B站官方指纹 API 获取真实 `b_3`/`b_4`，失败则生成随机 `buvid3`
  - B站请求自动注入 `Origin` 和 `Referer` 头
- **修复文件**：
  - `yt-dlp-fix/yt_dlp/extractor/bilibili.py`
  - `src/video_agent_skill/core/extractor.py`
- **验证情况**：
  - 手动测试 4 个不同的 B站视频（BV1GJ411x7h7、BV1XDLn6XEWK、BV1AbWCejEkn、BV13x41117TL），全部成功解析
  - `uv run pytest tests/ -q` 通过，58 个测试全部通过
  - `uv run ruff check .` 通过
- **备注**：B站部分视频的字幕需要登录才能获取（`need_login_subtitle`），这是 B站 API 的限制，非代码问题
