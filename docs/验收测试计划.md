# Video-Agent-Skill 验收测试计划

版本：v1.0  
日期：2026-06-05  
适用阶段：MVP 实现、集成联调、发版前回归

## 1. 测试目标

本计划用于验证 Video-Agent-Skill 是否满足“Agent 可稳定调用”的核心目标。验收重点不是人工可读日志，而是命令行契约、降级策略、异常熔断、隐私边界和临时文件生命周期。

## 2. 测试范围

### 2.1 覆盖范围

- CLI 参数解析：`--url`、`--lang`、`--proxy`、`--keep-temp`。
- 媒体提取：字幕优先、无字幕时音频降级。
- ASR 转写：VAD 切片、SenseVoice 推理、标签清洗。
- LLM 摘要：强制 JSON 输出、摘要长度、关键点数量。
- I/O 隔离：`stdout` 仅输出业务 JSON，日志全部进入 `stderr`。
- 异常处理：网络、鉴权、GPU OOM、LLM 超时。
- 清理机制：默认删除 UUID 临时工作目录，调试模式保留。

### 2.2 暂不覆盖

- 平台反爬策略的长期对抗。
- 付费或未授权内容的绕过。
- 大规模并发调度系统。
- GUI 或浏览器插件体验。

## 3. 测试环境

| 项目 | 要求 |
| --- | --- |
| Python | 3.10+ |
| 包管理 | `uv` |
| 媒体工具 | `FFmpeg` 可执行文件位于 `PATH` |
| ASR | `SenseVoiceSmall` 可在本地加载 |

## 5. 核心测试用例

| 编号 | 场景 | 输入 | 预期结果 |
| --- | --- | --- | --- |
| AT-001 | 带字幕视频 | 含中文字幕的公开视频 | `strategy_used=subtitle`，5 秒内返回 JSON |
| AT-002 | 无字幕视频 | 无字幕短视频 | `strategy_used=asr`，完成音频下载、切片、转写、摘要 |
| AT-003 | stdout 契约 | 将 stdout 重定向到文件 | 文件内容可被 `json.loads` 直接解析 |
| AT-004 | stderr 隔离 | 开启下载和模型日志 | 进度条、警告、Traceback 不进入 stdout |
| AT-005 | 代理路由 | YouTube URL + 代理规则 | 解析请求使用匹配域名的代理 |
| AT-006 | 鉴权失败 | 私有、付费或登录受限视频 | 返回 `AUTH_REQUIRED_ERROR`，退出码为 1 |
| AT-007 | 网络超时 | 断网或无效代理 | 返回 `PROXY_TIMEOUT` 或 `NETWORK_ERROR` |
| AT-008 | LLM 超时 | 模拟 API 超时 | 返回 `LLM_TIMEOUT`，无原生异常外泄 |
| AT-009 | 临时目录清理 | 默认模式运行 | 进程结束后本次 UUID 目录不存在 |
| AT-010 | 调试保留 | 使用 `--keep-temp` | 保留字幕、音频或切片文件，输出路径写入 `stderr` |
| AT-011 | Prompt 可配置 | 使用 `--llm-system-prompt-file` / `--llm-user-prompt-file` | 自定义 prompt 生效，stdout 仍为标准 JSON |
| AT-012 | Prompt 安装资源 | 使用 `--prompt-info` / `--init-prompts` | 可查看安装包内默认 prompt，并复制为本地可编辑文件 |
| AT-013 | Markdown 输出 | 使用 `--output-format markdown` | stdout 输出 Markdown 格式文本，包含标题、摘要、关键要点等章节 |
| AT-014 | 文件输出 | 使用 `--output-file result.json` | 结果写入指定文件，stdout 无输出 |
| AT-015 | Markdown 文件输出 | `--output-format markdown --output-file summary.md` | 生成结构化 Markdown 文件，可被 Markdown 阅读器正确渲染 |
| AT-016 | B站 412 绕过 | B站视频 URL | 通过动态 buvid3/buvid4 cookie 成功提取，不返回 HTTP 412 错误 |

## 6. JSON 契约断言

成功响应必须满足：

- `status` 等于 `success`。
- `meta.url` 与输入 URL 一致。
- `meta.strategy_used` 只能是 `subtitle`、`asr` 或后续扩展枚举。
- `content.summary` 非空且少于 200 个中文字符。
- `content.key_points` 为 3 到 5 条。
- `error` 为 `null`。

失败响应必须满足：

- `status` 等于 `error`。
- `content` 为 `null`。
- `error.code` 非空，且属于已登记错误码。
- `error.message` 面向 Agent 可理解，不包含本机绝对路径、密钥或完整 Traceback。

## 7. 错误码基线

| 错误码 | 含义 |
| --- | --- |
| `INVALID_ARGUMENT` | CLI 参数缺失或格式错误 |
| `UNSUPPORTED_URL` | 当前平台或链接格式不支持 |
| `AUTH_REQUIRED_ERROR` | 视频需要登录、会员、购买或权限 |
| `NETWORK_ERROR` | 网络不可达、DNS 失败或连接中断 |
| `PROXY_TIMEOUT` | 代理不可用或请求超时 |
| `SUBTITLE_PARSE_ERROR` | 字幕下载成功但解析失败 |
| `ASR_RUNTIME_ERROR` | ASR 模型加载或推理失败 |
| `CUDA_OOM_ERROR` | GPU 显存不足 |
| `LLM_TIMEOUT` | 摘要模型请求超时 |
| `LLM_SAFETY_REFUSAL` | LLM 服务因安全策略拒绝请求或输出 |
| `OUTPUT_CONTRACT_ERROR` | 最终输出无法序列化为标准 JSON |

## 8. 通过标准

MVP 发版前必须满足：

- P0 用例 AT-001 至 AT-009 全部通过。
- stdout 契约测试在 Windows PowerShell、Linux Shell 中均通过。
- 至少完成一个 B站 ASR fallback 样例和一个 YouTube 字幕样例的端到端验证。
- B站元数据中仅出现 `danmaku` 时，不视为可用 transcript 字幕，必须继续降级到 ASR。
- 临时文件清理在正常退出和异常退出路径均通过。
- README、需求规格和错误码表与实际实现保持一致。
