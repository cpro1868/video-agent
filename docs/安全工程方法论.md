# 安全工程方法论：防止敏感信息泄露

> 本文档提炼自 Video-Agent-Skill 项目 SEC-001 安全事件的经验教训，形成可复用于任何软件开发项目的安全编码规范和操作流程。

---

## 核心原则

### 黄金法则

> **绝不将任何敏感信息以明文形式提交到版本控制系统**

敏感信息包括但不限于：
- API Keys、Access Tokens、Secret Keys
- 密码、凭据、证书
- 私有 Cookie、Session Tokens
- 数据库连接字符串（含用户名密码）
- 完整的本地路径（可能暴露用户名）
- 下载的受限媒体内容
- 任何含有个人隐私信息的文件

## 预防体系

### 1. 防御层：.gitignore 规范

每个项目必须在仓库根目录配置 `.gitignore`，覆盖以下模式：

```gitignore
# ====================
# 敏感配置文件
# ====================
config.yaml
config.local.yaml
config.secrets.yaml
*.local.yaml
*.secret.yaml
.env
.env.*
.secrets

# ====================
# AI Agent 配置目录
# ====================
.claude/
.codex/
.agent/
.ai/

# ====================
# 用户运行时配置
# ====================
~/.project-name/           # Linux/macOS
%APPDATA%/project-name/    # Windows

# ====================
# IDE 和编辑器配置（可能含敏感信息）
# ====================
.idea/
.vscode/settings.json
*.sublime-*

# ====================
# 常见的临时文件
# ====================
*.log
*.tmp
*.temp
```

### 2. 防御层：Pre-commit Hook

#### 快速部署版（Shell Script）

```bash
#!/bin/sh
# .githooks/pre-commit
# 放置在项目 .githooks/ 目录

echo "🔍 Scanning for secrets..."

# 敏感模式定义
PATTERNS="sk-[a-zA-Z0-9_-]{20,}"
PATTERNS="$PATTERNS|api[_-]?key['\":\\s]*['\"][a-zA-Z0-9_-]{20,}"
PATTERNS="$PATTERNS|-----BEGIN [A-Z]+ PRIVATE KEY-----"
PATTERNS="$PATTERNS|ghp_[a-zA-Z0-9]{36}"
PATTERNS="$PATTERNS|gho_[a-zA-Z0-9]{36}"
PATTERNS="$PATTERNS|ghu_[a-zA-Z0-9]{36}"
PATTERNS="$PATTERNS|ghs_[a-zA-Z0-9]{36}"
PATTERNS="$PATTERNS|ghr_[a-zA-Z0-9]{36}"

FOUND_SECRETS=0

for file in $(git diff --cached --name-only); do
    if [ -f "$file" ] && [ -r "$file" ]; then
        if grep -E "$PATTERNS" "$file" 2>/dev/null; then
            echo "❌ Secret detected in: $file"
            FOUND_SECRETS=1
        fi
    fi
done

if [ $FOUND_SECRETS -eq 1 ]; then
    echo "Commit aborted. Remove secrets before committing."
    exit 1
fi

echo "✅ No secrets detected."
exit 0
```

#### 启用方式

```bash
# 启用项目的 git hooks 目录
git config core.hooksPath .githooks

# 确保 hook 可执行
chmod +x .githooks/pre-commit
```

#### 进阶版（使用 gitleaks）

```bash
#!/bin/bash
# 需要先安装: brew install gitleaks 或 pip install gitleaks

gitleaks protect --staged --verbose --cfg .gitleaks.toml
```

### 3. 防御层：GitHub Secret Scanning

#### 启用步骤

1. 进入 GitHub 仓库 → Settings → Security → Secret scanning
2. 启用 "Secret scanning" 和 "Push protection"
3. GitHub 会自动扫描推送到仓库的 secrets，并在推送时阻止包含敏感信息的提交

#### 自定义模式（如需要）

```yaml
# .github/secret_scanning/custom_patterns.yml
patterns:
  - name: Custom API Key Pattern
    pattern: "apikey-[a-zA-Z0-9]{32}"
    path: "**/*.{yaml,yml,json,env}"
```

### 4. 防御层：CI/CD 检查

#### GitHub Actions 示例

```yaml
# .github/workflows/secrets-scan.yml
name: Secrets Detection

on: [push, pull_request]

jobs:
  gitleaks:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      
      - name: Run gitleaks
        uses: gitleaks/gitleaks-action@v2
        env:
          GITLEAKS_LICENSE: ${{ secrets.GITLEAKS_LICENSE }}
```

#### GitLab CI 示例

```yaml
# .gitlab-ci.yml
secrets-detection:
  stage: pre-build
  script:
    - wget https://github.com/gitleaks/gitleaks/releases/download/v8.18.0/gitleaks_8.18.0_linux_x64.tar.gz
    - tar -xzf gitleaks_8.18.0_linux_x64.tar.gz
    - ./gitleaks detect --source . --verbose
  rules:
    - if: '$CI_PIPELINE_SOURCE == "push"'
```

### Git 历史清理命令参考

#### 使用 git filter-repo（推荐）

```bash
# 1. 安装工具
pip install git-filter-repo

# 2. 克隆一个新副本（不在原始仓库执行）
git clone https://github.com/user/repo.git
cd repo

# 3. 清理特定文件
git filter-repo --path config.yaml --invert-paths --force

# 4. 验证清理结果
git log --all -p | grep -E "sk-|api_key"  # 应无输出

# 5. 添加新的 remote（如果已有 origin 需要清理）
git remote add origin https://github.com/user/repo.git

# 6. 强制推送
git push --force --all
git push --force --tags
```

#### 使用 git filter-branch（备选）

```bash
# 1. 备份
cp -r .git /tmp/git_backup/

# 2. 重写历史
git filter-branch --force --index-filter \
  "git rm --cached --ignore-unmatch config.yaml" \
  --prune-empty --tag-name-filter cat -- --all

# 3. 清理 reflog
git reflog expire --expire=now --all
git gc --prune=now --aggressive

# 4. 推送
git push --force --all
```

## 配置管理最佳实践

### 1. 配置文件模板模式

```
project/
├── config.example.yaml      # 模板，包含所有字段，值为空或占位符
├── config.yaml              # 不提交，仅本地使用
└── .gitignore               # 排除 config.yaml
```

**config.example.yaml 示例**：

```yaml
ai:
  llm:
    api_base: ""                    # 留空，用户填入
    api_key: ""                     # 留空，用户填入
    model_name: ""                  # 留空，用户填入
```

### 2. 环境变量注入模式

```python
# config.py
import os

def load_config():
    api_key = os.environ.get("LLM_API_KEY", "")
    return {"api_key": api_key}
```

```bash
# 使用时
export LLM_API_KEY="sk-xxx"
python main.py
```

## 常见敏感模式速查表

| 类型 | 模式示例 | 检测正则 |
|------|---------|---------|
| OpenAI API Key | `sk-...` | `sk-[a-zA-Z0-9_-]{20,}` |
| GitHub Token | `ghp_...` | `ghp_[a-zA-Z0-9]{36}` |
| AWS Access Key | `AKIA...` | `AKIA[0-9A-Z]{16}` |
| Private Key | `-----BEGIN...` | `-----BEGIN [A-Z]+ PRIVATE KEY-----` |
| Generic API Key | `api_key: "..."` | `api[_-]?key['\":\s]*['\"][^'\"]{20,}` |
| Connection String | `mongodb://...` | `mongodb://[^@]+@[^:]+:\d+` |
| Bearer Token | `Authorization: Bearer ...` | `Bearer [a-zA-Z0-9_-]{20,}` |

## 项目自检清单

### 新项目初始化

- [ ] 创建 `.gitignore`，覆盖所有敏感文件模式
- [ ] 创建 `config.example.yaml` 或 `config.template.yaml`
- [ ] 配置 `.githooks/pre-commit` hook
- [ ] 启用 GitHub Secret Scanning
- [ ] 添加 CI secrets 检测步骤
- [ ] 团队安全培训：绝不提交敏感信息

### 每次 Commit 前

- [ ] 检查是否有新的敏感文件被创建
- [ ] 运行 `git diff --cached` 确认暂存内容
- [ ] pre-commit hook 通过（如果有）

### 每次 Release 前

- [ ] 全仓库扫描敏感模式
- [ ] 检查构建产物（wheel/jar/二进制）是否包含敏感信息
- [ ] 验证 `.gitignore` 覆盖所有需要排除的文件
